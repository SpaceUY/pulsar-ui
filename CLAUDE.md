# CLAUDE.md — Project Instructions for AI-Assisted Development

## Project Overview

`@space-uy/pulsar-ui` is a React Native UI kit library for SpaceDev. It is a monorepo using Yarn workspaces with the library at root and an example app in `example/`.

## Key Commands

- `yarn` — Install dependencies
- `yarn typecheck` — Run TypeScript type checking
- `yarn lint` — Run ESLint
- `yarn lint --fix` — Auto-fix lint issues
- `yarn test` — Run Jest unit tests
- `yarn prepare` — Build the library (bob build)
- `yarn example start` — Start Metro for the example app
- `yarn release` — Publish a new version via release-it

## Code Conventions

- **Language**: TypeScript (strict). All source files in `src/`.
- **Formatting**: Prettier — single quotes, 2-space indent, trailing commas (es5), no tabs.
- **Linting**: ESLint with `@react-native` config + Prettier plugin.
- **Commits**: Follow [Conventional Commits](https://www.conventionalcommits.org/en). Prefixes: `feat`, `fix`, `refactor`, `docs`, `test`, `chore`.
- **Branch naming**: Use `feat/`, `fix/`, `docs/`, `chore/`, `refactor/`, or `test/` prefixes (e.g., `feat/new-button-variant`).
- **PR titles**: Must follow conventional commits format (enforced by CI).

## Architecture

- `src/components/` — UI components
- `src/hooks/` — Custom React hooks
- `src/store/` — Zustand state management
- `src/theme/` — Theme definitions and tokens
- `src/utils/` — Utility functions
- `example/` — Expo example app for testing

## Versioning

This project follows [Semantic Versioning](https://semver.org/):
- **MAJOR** (x.0.0): Breaking API changes
- **MINOR** (0.x.0): New features, backward-compatible
- **PATCH** (0.0.x): Bug fixes, backward-compatible

Releases are managed via `release-it` with conventional changelog.

## Dependencies

- Peer dependencies (must be provided by consumers): React, React Native, Reanimated, Gesture Handler, SVG, Zustand, Lucide icons, React Navigation Elements.
- Do NOT add peer dependencies as regular dependencies.

## Testing

- Run `yarn test` before submitting changes.
- Run `yarn typecheck` and `yarn lint` to ensure code quality.
- Pre-commit hooks (lefthook) automatically run lint and typecheck on staged files.

## Things to Avoid

- Do not modify `lib/` — it is auto-generated by bob build.
- Do not commit `.env` files or secrets.
- Do not use `npm` — this project uses Yarn workspaces.
- Do not bypass pre-commit hooks with `--no-verify`.
