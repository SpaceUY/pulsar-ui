# CLAUDE.md ‚Äî Project Instructions for AI-Assisted Development

## Project Overview

`@space-uy/pulsar-ui` is a React Native UI kit library for SpaceDev. It is a monorepo using Yarn workspaces with the library at root and an example app in `example/`.

## Key Commands

- `yarn` ‚Äî Install dependencies
- `yarn typecheck` ‚Äî Run TypeScript type checking
- `yarn lint` ‚Äî Run ESLint
- `yarn lint --fix` ‚Äî Auto-fix lint issues
- `yarn test` ‚Äî Run Jest unit tests
- `yarn prepare` ‚Äî Build the library (bob build)
- `yarn example start` ‚Äî Start Metro for the example app
- `yarn release` ‚Äî Publish a new version via release-it

## Code Conventions

- **Language**: TypeScript (strict). All source files in `src/`.
- **Formatting**: Prettier ‚Äî single quotes, 2-space indent, trailing commas (es5), no tabs.
- **Linting**: ESLint with `@react-native` config + Prettier plugin.
- **Commits**: Follow [Conventional Commits](https://www.conventionalcommits.org/en). Prefixes: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`, `revert`.
- **Branch naming**: Use `feat/`, `fix/`, `docs/`, `style/`, `refactor/`, `perf/`, `test/`, `build/`, `ci/`, `chore/`, or `revert/` prefixes (e.g., `feat/new-button-variant`). Enforced by a pre-push hook.
- **PR titles**: Must follow conventional commits format (enforced by CI).

## Architecture

- `src/components/` ‚Äî UI components
- `src/hooks/` ‚Äî Custom React hooks
- `src/store/` ‚Äî Zustand state management
- `src/theme/` ‚Äî Theme definitions and tokens
- `src/utils/` ‚Äî Utility functions
- `example/` ‚Äî Expo example app for testing

## Versioning

This project follows [Semantic Versioning](https://semver.org/):
- **MAJOR** (x.0.0): Breaking API changes
- **MINOR** (0.x.0): New features, backward-compatible
- **PATCH** (0.0.x): Bug fixes, backward-compatible

Releases are managed via `release-it` with conventional changelog.

## Dependencies

- Peer dependencies (must be provided by consumers): React, React Native, Reanimated, Gesture Handler, SVG, Zustand, Lucide icons, React Navigation Elements.
- Do NOT add peer dependencies as regular dependencies.

## Testing

- Run `yarn test` before submitting changes.
- Run `yarn typecheck` and `yarn lint` to ensure code quality.
- Pre-commit hooks (lefthook) automatically run lint and typecheck on staged files.

## Things to Avoid

- Do not modify `lib/` ‚Äî it is auto-generated by bob build.
- Do not commit `.env` files or secrets.
- Do not use `npm` ‚Äî this project uses Yarn workspaces.
- Do not bypass pre-commit hooks with `--no-verify`.
- Never author or co-author commits. Do not add `Co-Authored-By: Claude ...` or similar AI attribution lines to commit messages, PR descriptions, or CHANGELOG.md.
